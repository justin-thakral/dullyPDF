# Code map (what each module does)

This file is a quick guide to the sandbox codebase layout and how the modules fit together.

## Directory layout

- `backend/combinedSrc/`: Shared preprocessing + utilities used by both pipelines.
- `backend/nativeSrc/`: Native/text-layer resolver pipeline.
- `backend/scannedSrc/`: Scanned pipeline.
- `backend/debug/`: CLI tooling for local runs and QA overlays.
- `backend/tools/`: Maintenance/batch scripts (sorting, migration, etc.).
- `backend/test/`: Unit tests for core geometry/coordinate logic.
- `backend/forms/`: PDFs generated by sandbox tooling.
- `backend/outputArtifacts/`: Default output location (`json/` and `overlays/`).
- `backend/docs/images/`: Reference screenshots used when tuning heuristics/prompts.
- `backend/docs/tools.md`: Reference for maintenance scripts.

## Core data contracts

### Candidates (detection output)

Candidates are stored per page with stable IDs:
- `line-<page>-<n>`: underline candidates (text inputs)
- `checkbox-<page>-<n>`: checkbox square candidates
- `box-<page>-<n>`: rectangle candidates (signature/date/table cells)

The main assembly happens in `backend/combinedSrc/build_candidates.py`.

### Fields (resolver output)

Fields are what downstream injectors consume:
- `name`: final field name
- `type`: `text | checkbox | signature | date`
- `page`: 1-based
- `candidateId`: which candidate the field is backed by
- `rect`: computed deterministically from the candidate
- `confidence`: model/heuristic confidence

Rect building rules are centralized in `backend/combinedSrc/rect_builder.py`.

## Execution flow (end-to-end)

The orchestrator is `backend/combinedSrc/pipeline_router.py`.

It does:
1) `render_pdf_to_images` (`backend/combinedSrc/render_pdf.py`)
2) `detect_geometry` (`backend/combinedSrc/detect_geometry.py`)
3) `extract_labels` (`backend/combinedSrc/extract_labels.py`)
4) `assemble_candidates` (`backend/combinedSrc/build_candidates.py`)
5) `compute_label_height_calibration` (`backend/combinedSrc/calibration.py`)
6) Route to native or scanned pipeline

## Key modules (combinedSrc)

- `backend/combinedSrc/config.py`
  - Sandbox config + `.env` bootstrap + logging toggle (`SANDBOX_DEBUG`).

- `backend/combinedSrc/render_pdf.py`
  - Renders PDF pages to images at `SANDBOX_DPI`.

- `backend/combinedSrc/coords.py`
  - Converts between pixel bboxes and point bboxes and handles rotation/origin conventions.

- `backend/combinedSrc/detect_geometry.py`
  - OpenCV geometry detection for lines/boxes/checkboxes.
  - Emits bboxes in point space (originTop) and includes pixel bboxes for debug.

- `backend/combinedSrc/extract_labels.py`
  - Extracts prompt/label text, using OCR fallback (`backend/combinedSrc/ocr_labels.py`) when needed.

- `backend/combinedSrc/label_index.py`
  - Spatial index over label midpoints to speed up “nearest label to candidate” lookups.

- `backend/combinedSrc/heuristic_resolver.py`
  - Deterministic resolver for both pipelines.
  - Focuses on high recall and stable geometry; naming is best-effort.

- `backend/combinedSrc/openai_utils.py`
  - OpenAI Responses API helpers:
    - temperature fallback (some models reject `temperature`)
    - robust `output_text` extraction

- `backend/combinedSrc/vision_utils.py`
  - Shared image helpers (encode to data URLs, crop with padding).

- `backend/combinedSrc/field_overlay.py`
  - Produces an overlay image with candidate boxes and field labels for LLM input.

- `backend/combinedSrc/rename_resolver.py`
  - Page-level rename resolver:
    - renders one full-page overlay per page
    - calls OpenAI with a strict line-based format
    - returns renamed fields with confidence gating

- `backend/combinedSrc/output_layout.py`
  - Standardizes output layout under `json/` + `overlays/` and provides temp prefixes.

- `backend/combinedSrc/form_filler.py`
  - Sandbox-only PDF injection tool that writes actual AcroForm fields into a PDF.
  - Not used by the production pipeline; intended for offline debugging and prototyping.

## Native pipeline (nativeSrc)

- `backend/nativeSrc/pipeline.py`
  - Entry point for native pipeline; deterministic heuristics only.

## Scanned pipeline (scannedSrc)

- `backend/scannedSrc/pipeline.py`
  - Entry point for scanned pipeline; deterministic heuristics only.

## Debug tooling (debug)

- `backend/debug/test_local.py`
  - Primary CLI for local runs; writes `temp<first5><last5>_fields.json`, `temp<first5><last5>_candidates.json`, and can run:
    - `--openAI`

- `backend/debug/debug_overlay.py`
  - Draws QA overlays (candidates and final fields) for human inspection.

## Maintenance tools (tools)

- `backend/tools/sort_pdfs.py`
  - Classifies PDFs into `pdfs/native` and `pdfs/scanned` (and optionally form PDFs).

- `backend/tools/migrate_ml_pdfs.py`
  - Moves ML corpus PDFs into the `pdfs/native|scanned` layout.

- `backend/tools/migrate_analysis_artifacts.py`
  - Migrates legacy `analysis_*` runs into `outputArtifacts/json` + `outputArtifacts/overlays`.

- `backend/tools/artifactManager.py`
  - Cleans temp-prefixed artifacts and prunes empty directories.
